# Data Structures

Practice implementations of common data structures in C++, from basics to advanced.

---

## Linear Structures

### Array
| | |
|---|---|
| **优点** | 随机访问 O(1)；内存连续，缓存友好；实现简单 |
| **缺点** | 固定大小；插入/删除 O(n)（需移动元素） |
| **使用场景** | 元素数量固定、频繁随机访问；作为其他结构的底层存储（堆、哈希表） |

### Vector（动态数组）
| | |
|---|---|
| **优点** | 随机访问 O(1)；自动扩容；尾部插入均摊 O(1) |
| **缺点** | 扩容时有一次 O(n) 拷贝；中间插入/删除 O(n) |
| **使用场景** | 最常用的序列容器；元素数量不确定但需要随机访问；替代绝大多数数组场景 |

### List（链表）
| | |
|---|---|
| **优点** | 任意位置插入/删除 O(1)（已有迭代器时）；无需连续内存；大小动态 |
| **缺点** | 随机访问 O(n)；额外指针开销；缓存不友好 |
| **使用场景** | 频繁在中间插入/删除；实现 LRU Cache、任务队列；不需要随机访问的场景 |

### Stack（栈）
| | |
|---|---|
| **优点** | 压栈/弹栈 O(1)；语义清晰，强制 LIFO |
| **缺点** | 只能访问栈顶；不支持随机访问 |
| **使用场景** | 函数调用栈；表达式求值/括号匹配；DFS 迭代实现；撤销（Undo）功能 |

### Queue（队列）
| | |
|---|---|
| **优点** | 入队/出队 O(1)；语义清晰，强制 FIFO |
| **缺点** | 只能访问队首/队尾；不支持随机访问 |
| **使用场景** | BFS；任务调度；消息队列；生产者-消费者缓冲 |

### Deque（双端队列）
| | |
|---|---|
| **优点** | 两端插入/删除均 O(1)；支持随机访问 |
| **缺点** | 内存不连续（分块存储），缓存效果差于 vector |
| **使用场景** | 滑动窗口最大值（单调队列）；需要两端操作的队列；`std::deque` 是 `std::stack`/`std::queue` 的默认底层容器 |

## Tree Structures

### Binary Tree（二叉树）
| | |
|---|---|
| **优点** | 结构灵活；层序/前序/中序/后序遍历覆盖大量问题 |
| **缺点** | 无序，查找 O(n)；退化链表时所有操作 O(n) |
| **使用场景** | 表达式树；Huffman 编码；作为 BST/AVL/红黑树的基础 |

### BST（二叉搜索树）
| | |
|---|---|
| **优点** | 有序；平均查找/插入/删除 O(log n)；中序遍历得有序序列 |
| **缺点** | 最坏退化为链表 O(n)；不自平衡 |
| **使用场景** | 需要有序动态集合且数据基本随机时；理解平衡树的基础 |

### AVL Tree
| | |
|---|---|
| **优点** | 严格平衡（高度差 ≤ 1）；查找 O(log n) 最优 |
| **缺点** | 旋转频繁（插入/删除最多 O(log n) 次旋转）；写操作较慢 |
| **使用场景** | 读多写少的有序集合；对查找性能要求极高的场景 |

### Red-Black Tree（红黑树）
| | |
|---|---|
| **优点** | 近似平衡；插入/删除最多 3 次旋转；综合性能好 |
| **缺点** | 实现复杂；不如 AVL 严格平衡，查找略慢 |
| **使用场景** | C++ `std::map`/`std::set`；Java `TreeMap`；Linux 进程调度（CFS）；需要频繁增删的有序集合 |

### Trie（前缀树）
| | |
|---|---|
| **优点** | 字符串查找 O(m)（m 为串长）；天然支持前缀匹配 |
| **缺点** | 内存占用大（每节点 26 个子指针）；不适合非字符串键 |
| **使用场景** | 自动补全；拼写检查；IP 路由表（CIDR 前缀匹配）；词频统计 |

### Segment Tree（线段树）
| | |
|---|---|
| **优点** | 区间查询/更新 O(log n)；支持懒标记批量更新 |
| **缺点** | 空间 O(4n)；实现较复杂 |
| **使用场景** | 区间求和/最大值/最小值；区间赋值；动态 RMQ；竞赛算法核心结构 |

### Binary Indexed Tree / Fenwick Tree（树状数组）
| | |
|---|---|
| **优点** | 前缀求和/单点更新 O(log n)；实现极简；常量因子小 |
| **缺点** | 功能受限（仅支持前缀可逆操作）；不如线段树通用 |
| **使用场景** | 前缀和动态维护；逆序对计数；求第 k 小；能用线段树的场景优先考虑 BIT |

### Heap（堆）
| | |
|---|---|
| **优点** | 取最值 O(1)；插入/删除 O(log n)；原地建堆 O(n) |
| **缺点** | 不支持随机访问；查找任意元素 O(n) |
| **使用场景** | 优先队列；堆排序；Dijkstra/Prim 最短路；Top-K 问题；任务调度 |

## Hash Structures

### HashMap（哈希表）
| | |
|---|---|
| **优点** | 查找/插入/删除均摊 O(1)；灵活键类型 |
| **缺点** | 哈希冲突导致最坏 O(n)；无序；额外内存开销 |
| **使用场景** | 词频统计；缓存（key-value）；去重；两数之和等索引查找问题 |

### HashSet（哈希集合）
| | |
|---|---|
| **优点** | 成员查询 O(1)；去重高效 |
| **缺点** | 无序；不支持范围查询 |
| **使用场景** | 快速判断元素是否存在；集合求交/并/差；访问记录去重 |

### Bloom Filter（布隆过滤器）
| | |
|---|---|
| **优点** | 空间极省（位数组）；查询 O(k)；无假阴性 |
| **缺点** | 有假阳性（可调但不可消除）；标准版不支持删除 |
| **使用场景** | 缓存穿透防护；爬虫 URL 去重；数据库查询前置过滤；垃圾邮件检测 |

### Consistent Hash（一致性哈希）
| | |
|---|---|
| **优点** | 节点增删只影响相邻节点的少量数据；虚拟节点解决负载不均 |
| **缺点** | 实现比普通取模复杂；虚拟节点增加内存开销 |
| **使用场景** | 分布式缓存（Redis Cluster）；CDN 节点路由；分布式存储数据分片 |

## Graph Structures

### Adjacency Matrix（邻接矩阵）
| | |
|---|---|
| **优点** | 判断两点是否相邻 O(1)；实现简单 |
| **缺点** | 空间 O(V²)；稀疏图浪费严重；遍历所有边 O(V²) |
| **使用场景** | 稠密图（边数接近 V²）；Floyd 全源最短路；需要频繁查询边权 |

### Adjacency List（邻接表）
| | |
|---|---|
| **优点** | 空间 O(V+E)；遍历所有边 O(V+E)；适合稀疏图 |
| **缺点** | 判断两点是否相邻 O(degree)；实现略复杂 |
| **使用场景** | 绝大多数图算法（BFS/DFS/Dijkstra/拓扑排序）；稀疏图标准表示 |

### Union-Find（并查集）
| | |
|---|---|
| **优点** | 合并/查找近乎 O(1)（反阿克曼函数）；实现极简 |
| **缺点** | 仅支持合并，不支持分裂；不能直接枚举集合成员 |
| **使用场景** | Kruskal 最小生成树；检测无向图环；动态连通性；朋友圈/社交网络分组 |

### Topological Sort（拓扑排序）
| | |
|---|---|
| **优点** | O(V+E) 线性时间；能检测有向环 |
| **缺点** | 仅适用于 DAG（有向无环图） |
| **使用场景** | 编译依赖顺序；任务调度；课程先修关系；构建系统（Make/CMake） |

## Advanced Structures

### Skip List（跳表）
| | |
|---|---|
| **优点** | 期望 O(log n) 查找/插入/删除；实现比红黑树简单；天然有序 |
| **缺点** | 空间 O(n log n)；依赖随机化，最坏退化；不如红黑树缓存友好 |
| **使用场景** | Redis ZSet（有序集合）底层；需要有序且并发友好的场景；替代平衡树 |

### Ring Buffer（环形缓冲区）
| | |
|---|---|
| **优点** | 固定内存，无动态分配；入队/出队 O(1)；缓存友好 |
| **缺点** | 容量固定，满时需策略处理（阻塞/覆盖/丢弃） |
| **使用场景** | 音视频流缓冲；日志循环覆盖写；内核网络收发缓冲区；SPSC 无锁队列 |

### Lock-Free Queue（无锁队列）
| | |
|---|---|
| **优点** | 无互斥锁，高并发吞吐高；避免死锁/优先级反转 |
| **缺点** | 实现复杂（ABA 问题、内存回收）；非极高并发时性能不一定优于有锁版 |
| **使用场景** | 高频交易/游戏引擎线程间通信；多核高并发任务队列；实时系统避免锁争用 |

### LRU Cache（最近最少使用缓存）
| | |
|---|---|
| **优点** | get/put O(1)；自动淘汰冷数据；符合局部性原理 |
| **缺点** | 额外指针内存开销；不适合扫描型访问（缓存污染） |
| **使用场景** | CPU/磁盘页面置换；浏览器/DNS 缓存；数据库 Buffer Pool；Redis 内存淘汰策略 |

### Time Wheel（时间轮）
| | |
|---|---|
| **优点** | 添加/取消定时器 O(1)；处理海量定时器时远优于最小堆 O(log n) |
| **缺点** | 精度受 tick 间隔限制；单层时间轮表示范围有限（需多层） |
| **使用场景** | TCP 超时重传；心跳检测；游戏技能 CD；Kafka 延迟消息；Netty HashedWheelTimer |

## 选型速查

| 需求 | 推荐结构 |
|---|---|
| 随机访问 + 动态大小 | Vector |
| 频繁中间插入/删除 | List |
| 有序集合，读多写少 | AVL Tree |
| 有序集合，读写均衡 | Red-Black Tree |
| 无序 key-value，极速查询 | HashMap |
| 字符串前缀匹配 | Trie |
| 区间查询/更新 | Segment Tree / BIT |
| 动态取最值 | Heap（优先队列）|
| 海量数据快速判存在 | Bloom Filter |
| 图的通用表示 | Adjacency List |
| 动态连通性 | Union-Find |
| 分布式数据分片 | Consistent Hash |
| 高并发线程间通信 | Lock-Free Queue / Ring Buffer |
| 大量定时任务 | Time Wheel |
| 带过期的 key-value 缓存 | LRU Cache |
